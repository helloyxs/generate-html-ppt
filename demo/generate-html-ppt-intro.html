CORE ARCHITECTURE
核心机制：16:9 像素级 固定舞台
基于 CSS transform 缩放，而非复杂的响应式重排，确保在任何屏幕上布局 100% 相同。
渲染机制

缩放 vs 重排

  • Fixed 1920x1080 Stage：内部尺寸全部使用固定像素（px）编写。
  • Auto Scale Fit：JS 监听窗口变化，计算最佳比例使用 transform: scale() 整体缩放。
  • Letterboxing：在非 16:9 屏幕上自动居中并留白边，绝不重排挤压。
graph TD
    Win[浏览器窗口 Resize] --> Calc[计算缩放比 scale = min W/1920, H/1080]
    Calc --> Apply[transform: translate... scale...]
    Apply --> View[100% 还原的 16:9 幻灯片]
这个项目的核心机制是“16:9 像素级固定舞台”。我们不是使用媒体查询去重排，而是用 JS 监听缩放，直接用 transform 进行缩放。这样就能保证无论在手机、平板还是大屏上，看到的画面都是像素级一致的。

单栏要点特征

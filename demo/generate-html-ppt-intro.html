01 / PAIN POINTS & BREAKTHROUGH

告别 AI 同质化与不可控全自动

传统 AI 演示文稿生成工具常陷入“审美同质化”与“一键生成不可控”的双重瓶颈。

01
🔴 传统 AI PPT 的两大痛点
  • 审美同质化疲劳：默认 Inter 字体、紫白渐变、千篇一律的仪表盘卡片，缺乏表达力。
  • 黑盒生成不可控：一键全自动不可预测，文案错乱或结构排版崩坏后难以二次调整。
  • 画面缩放错乱：在不同比例大屏上重新排版，导致图片文案重叠错位。
02
🟢 generate-html-ppt 的解法
  • Design Recipe 解耦：字体、色盘、阴影堆栈、微交互写成独立的 `design.md`。
  • 3 阶段透明协作：文本需求对齐 ➔ 灰度骨架确认 (Wireframe) ➔ 细节填充。
  • 16:9 原生画质：1920×1080 Canvas 整体等比缩放，绝不打乱版式。
02 / CORE ARCHITECTURE

design.md 规范与渐进式加载

将视觉设计规则书与代码解耦，实现极致审美表达与超高 Token 效率。

A
34+ 专属设计规范

覆盖 8-Bit 像素风、Swiss 国际主义、Emerald 典雅杂志、Cobalt 极客风等，每套包含专属 Google Fonts/Fontshare 字体组合与 HSL 色盘。

B
Progressive Disclosure

先读取轻量级 `selection-index.json` 索引匹配风格调性，仅在选中后单点加载单份 `design.md`，极大节省 Context Window。

C
自由版式编排

AI 在 Design Recipe 的规范与约束下自由组合 DOM 结构，彻底摆脱硬编码 HTML 组件编号（L01-L12）的刻板束缚。

03 / VISUAL & EXPERIENCE

16:9 原生 Stage 与视口无缝融合

1920×1080 像素级画质，在 21:9 超宽屏、16:10 及各类大屏设备上无缝呈现。

📐 Fixed 16:9 Canvas Stage
  • 所有 Slide 统一在 1920×1080 画布内精心设计。
  • 通过 JavaScript 实时计算 `transform: scale(...)` 整体均匀缩放。
  • 移动端或超宽屏设备上始终保持 16:9 比例，绝不重新排版错位。
🎨 Seamless Viewport Background
  • 页面包含 `--viewport-bg` CSS 变量与 `updateViewportBg()` 动态同步。
  • 外侧屏幕背景与当前 Slide 背景（渐变、深色或纸张纹理）自然融合。
  • 在 21:9 宽屏或 16:10 屏幕上，告别传统黑边/白边的突兀感受。
04 / WORKFLOW

人类级 3 阶段透明协作流程

把掌控权交还给创作者，确保每一页结构与文案都精准符合预期。

01
需求对齐 & 品牌嗅探

通过 7 问对话对齐场景、受众与篇幅，自动嗅探提取企业品牌色（VI HEX/RGB）并注入 CSS 变量。

02
灰度骨架确认 (Wireframe)

先生成包含 DOM 结构与 Design Tokens 的 HTML 骨架，暂停并等待创作者确认结构后再编写详细内容。

03
密度与细节填充

依据 Speaker-led（演讲型）或 Reading-first（阅读型）密度模式精细填充文案、数据与多媒体资产。

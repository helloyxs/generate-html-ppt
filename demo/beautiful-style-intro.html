Context & Comparison

传统 PPT 痛点 vs Beautiful 样式解法

打破传统 Web 演示文稿的僵硬与空白陷阱，重构专业级视觉品质

✕

传统 Web 演示文稿痛点

  • 中间中空断层：流式布局导致不同分辨率下中间大面积尴尬空白
  • 黑框割裂感：视口边缘与幻灯片画布背景色不匹配，边界生硬
  • 动效死板：使用线性渐变或机械位移，缺乏现代 App 的弹性回响
✓

Beautiful Modern 解决之道

  • Scheme 1/2 密度控制：紧凑分组与中间连通桥，彻底消除尴尬留白
  • Seamless Viewport Sync：视口背景与当前页沉浸式同步
  • Spring 弹性物理微交互：采用 60fps 弹簧曲线与微光悬浮
Design System Specifications

核心设计规范：字体、色彩与层次

严格的 Design Tokens 构建出高级、和谐且富有生命力的视觉基底

01

Typography 双核字体

Display: Outfit (800/700) 现代几何标题字体

Body: Plus Jakarta Sans / Noto Sans SC 中英文高清晰度正文搭配

02

Saturated Accent 调色盘

以靛蓝/青蓝色主渐变（Indigo-Cyan Gradient）为核心，辅以 Teal/Rose/Amber/Violet 四色标点，增强信息辨识度

03

Glassmorphism 毛玻璃

多层级 `backdrop-filter: blur(16px)`，搭配高细度微发光边框 (`rgba(255,255,255,0.12)`)，营造晶莹质感

Layout Density Strategy

防止中间尴尬中空：Scheme 1 vs Scheme 2

根据内容区块数量动态匹配最佳垂直空间分配法则，消除页面视觉稀疏与断层

1

2 区块场景 (Scheme 1)

  • 标题 + 卡片两组顶端对齐
  • 使用 `gap: 40px` 紧凑靠拢
  • 底部自然留出呼吸空间
2

3 区块场景 (Scheme 2)

  • 三层结构使用 `.between` 铺满
  • 中间插入 Pipeline 或 Stat 连通桥
  • 强行拉紧上下元素视觉关联
3

全视口无缝统一

  • JS 实时提取幻灯片背景底色
  • 动态同步屏幕视口 `--viewport-bg`
  • 彻底消除边缘黑框与视觉分界
